# Qrcoder - 二维码生成工具

一个本地运行的 Web 二维码生成工具。启动后自动打开浏览器，左侧输入文本或选择文件，右侧实时生成二维码。支持中文、日文、英文、换行，内容过长时自动分页。

## 项目简介

Qrcoder 是一个轻量级的本地二维码生成工具，基于 Flask 构建。它解决了将长文本编码为二维码的需求——当文本内容超过单个二维码容量时，工具会自动将文本切分为多个二维码，用户可以通过上一张/下一张按钮依次浏览。

### 功能特性

- **文本输入**：直接在文本框中输入内容生成二维码
- **文件上传**：选择本地文本文件（`.txt`、`.md`、`.csv`、`.json` 等）生成二维码
- **多语言支持**：中文、日文、英文、换行符，UTF-8 字符级切割确保不截断多字节字符
- **自动分页**：内容超长时自动切割为多个二维码（每块最大 1800 字节），按顺序编号
- **分页浏览**：上一张/下一张按钮切换，支持键盘 ← → 方向键
- **宽屏布局**：左侧输入、右侧二维码展示，充分利用 PC 宽屏；窄屏自动堆叠

## 环境要求

- **Python** ≥ 3.11
- **uv**（Python 包管理工具）— [安装指南](https://docs.astral.sh/uv/getting-started/installation/)

## 安装

### 1. 克隆项目

```bash
git clone <仓库地址>
cd Qrcoder
```

### 2. 安装依赖

使用 `uv` 自动创建虚拟环境并安装所有依赖：

```bash
uv sync
```

这会根据 `pyproject.toml` 和 `uv.lock` 安装以下依赖：

| 依赖 | 用途 |
|------|------|
| Flask | Web 框架 |
| qrcode[pil] | 二维码生成 |
| Pillow | 图像处理 |

## 启动

### 方式一：命令行脚本（推荐）

使用项目根目录下的 `qrcoder.sh` 脚本，自动处理端口检查、依赖安装和启动：

```bash
# 直接启动，打开浏览器
./qrcoder.sh

# 启动并自动生成二维码（文本内容）
./qrcoder.sh -m "世界你好"

# 启动并读取文件内容生成二维码（相对路径）
./qrcoder.sh -f test.txt

# 启动并读取文件内容生成二维码（绝对路径）
./qrcoder.sh -f /c/x/d/test.txt
```

脚本会自动执行以下操作：
1. 检查 5000 端口占用情况，如被占用则先关闭旧进程
2. 运行 `uv sync` 安装依赖
3. 启动 Flask 服务并自动打开浏览器
4. 如果带 `-m` 或 `-f` 参数，页面加载后自动生成二维码

### 方式二：手动启动

```bash
uv sync
uv run python main.py
```

启动后：

1. 终端会显示 Flask 运行日志
2. **自动打开浏览器**，访问 `http://127.0.0.1:5000`
3. 如果浏览器没有自动打开，手动访问上述地址即可

## 使用方法

### 文本输入模式

1. 点击顶部「文本输入」标签
2. 在文本框中输入要生成二维码的内容
3. 点击「生成二维码」按钮
4. 右侧区域显示二维码图片

### 文件上传模式

1. 点击顶部「文件上传」标签
2. 点击「选择文本文件」按钮，选择本地文件
3. 文件内容会自动读取并预览
4. 点击「生成二维码」按钮

### 多二维码分页

当输入内容超过单个二维码容量（约 1800 字节）时：

- 工具自动将文本切分为多个二维码
- 页面顶部提示分为几张二维码
- 使用「上一张」「下一张」按钮切换，或键盘 ← → 方向键
- 页面底部显示当前页码（如「第 2 / 5 张」）和字符数

### 支持的文件类型

`.txt`、`.text`、`.md`、`.csv`、`.json`、`.log`、`.xml`、`.html`、`.yaml`、`.yml`

### 输入限制

最大支持 100 KB 文本内容。

## 停止运行

在运行 `uv run python main.py` 的终端窗口中，按下 `Ctrl + C` 即可停止服务。

如果服务器在后台运行，可以通过以下方式停止：

```bash
# 查找占用 5000 端口的进程
lsof -i:5000

# 终止进程（将 <PID> 替换为实际进程 ID）
kill <PID>
```

## 常见问题

**Q: 启动后浏览器没有自动打开？**

手动在浏览器中访问 `http://127.0.0.1:5000`。

**Q: 端口 5000 被占用怎么办？**

修改 `main.py` 最后两行中的端口号，例如改为 `5001`：

```python
threading.Timer(1.0, lambda: webbrowser.open("http://127.0.0.1:5001")).start()
app.run(host="127.0.0.1", port=5001, debug=False)
```

**Q: 生成的二维码扫描后内容不完整？**

内容可能被分成了多个二维码，请扫描所有二维码并手动拼接文本。每张二维码的页码显示在右下方。

**Q: 支持加密吗？**

当前版本不支持加密，二维码内容为原始文本。
